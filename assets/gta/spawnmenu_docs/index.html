Hop_SpawnMenu

Thank you for purchasing my resource, all the information you should need for setup/config should be available in this document but if you need more help please contact me on discord: hop#8403.

Installation
  • Place hop_spawnmenu in your resources folder.
  • Add start hop_spawnmenu to your server.cfg

Setup

Setup can take some time depending on how your server operates, if you encounter any issues feel free to contact me.

ESX Multi Character

One line of code must be added to allow the spawnmenu to open at the right time.

esx_multicharacter/client/main.lua

  • Line 296/297: Must be commented out (add "--" infront)
  • Line 298: TriggerEvent('hop_spawnmenu:setNui')

ESX Single Character

For single character ESX, 2 lines of code must be added to lua files within the esx_identity and esx_skin resources.

esx_identity/client/main.lua

  • Line 5: TriggerEvent("hop_spawnmenu:setNui")

esx_skin/client/main.lua

  • Line 234: TriggerEvent("hop_spawnmenu:setNui")

Standalone

Ensure Config.isStandAlone is set to true in the config.lua file.

Locations

Commands
  • spawnmenu: Opens the spawnmenu
  • spawnmenulocation: Prints your current vector3 coords in console
  • spawnmenudebug: Creates a placeable marker on the map and gives the user the X and Y Coords of the marker.

Seamless Loading

ESX

To acheive seamless loading with ESX, add loadscreen_manual_shutdown "yes" to the fxmanifest.lua of your loading screen resource. ESX will then terminate the loading screen automatically at the correct time.

Standalone

To acheive seamless loading on standalone, follow this tutorial initially.
After that, replace all the code in the new client.lua with:

RegisterNetEvent("loading_screen:endLoad")
AddEventHandler("loading_screen:endLoad", function()
ShutdownLoadingScreenNui()
end)

Finally, ensure Config.manualLoadEnabled is set to true.
